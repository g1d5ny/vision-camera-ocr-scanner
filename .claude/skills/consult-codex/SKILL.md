# 큰 결정은 Codex와 상의

## 언제 (이럴 땐 혼자 정하지 말 것)

- **아키텍처** 결정 — Nitro vs Expo, 모듈 구조, 스레딩 모델 등
- **의존성** 추가/교체 — vision-camera 버전, 새 라이브러리 도입
- **공개 API 설계** — Nitro 스펙 형태, 스캔 모드 인터페이스
- **범위 변경** — 스캔 모드 추가/삭제, 파괴적 변경(breaking change), 배포 전략

> 작고 되돌리기 쉬운 변경은 상의 없이 그냥 진행한다. 이 규칙은 "큰 변화/중요 결정"에만 적용.

## 어떻게

1. 결정 사항과 선택지를 **트레이드오프 포함**해 간단히 정리한다.
2. Codex에 비대화형으로 의견을 묻는다:
   ```bash
   codex exec "결정 맥락 + 선택지 A/B + 각 트레이드오프 + 구체적 질문"
   ```
3. **Codex 의견 + 내 분석을 종합**해 사용자에게 추천안을 제시하고, 최종 결정은 사용자가 한다.

## 실행 환경

- codex CLI: **codex-cli 0.142.3** (정상 작동). 위치 `~/.n/bin/codex` (Node 20).
- 이 repo 명령은 `~/.n/bin`을 PATH 앞에 두므로 이 codex가 쓰인다:
  ```bash
  export PATH="$HOME/.n/bin:$PATH"
  ```
- 최초 1회 인증이 필요할 수 있다(`codex login`). 인증이 안 돼 있어 `codex exec`가 실패하면, 그 사실을 사용자에게 알리고 정리한 결정 맥락을 전달해 사용자가 직접 Codex에 물어보게 한다.

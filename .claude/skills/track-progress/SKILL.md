# TODO.md로 진행상황 추적

repo 루트에 사람이 읽는 체크리스트 `TODO.md`를 유지한다. 이것이 사용자가 읽는 단일 진실 공급원(source of truth)이다.

## 형식

- `##` 단계(phase) 제목으로 작업을 묶는다 (예: "Setup", "MRZ mode").
- 각 작업은 체크박스:
  - `- [ ]` 시작 안 함
  - `- [~]` 진행 중
  - `- [x]` 완료
- 한 줄에 한 작업, 명령형 현재시제 ("MRZ 체크섬 검증 구현").

## 규칙

- 의미 있는 작업을 **시작**하면 `- [~]`로 표시.
- **완료**하면 `- [x]`로 바꾸고 목록에 남겨둔다(히스토리 삭제 금지).
- 새 작업이 생기면 알맞은 단계 아래에 `- [ ]`로 추가.
- 작업을 시작/완료한 **같은 턴**에 `TODO.md`를 갱신한다 — 절대 어긋나게 두지 않는다.
- 체크 후에는 무엇이 끝났고 다음이 무엇인지 사용자에게 보고한다.
- 가능하면 하니스 task 도구(TaskCreate/TaskUpdate)에도 미러링하되, `TODO.md`가 기준이다.
- **개인정보 금지** — `TODO.md`는 나중에 공개 git에 올라간다. 기기 이름, Apple 개발팀 이름/팀 ID, 서명 인증서, 계정/이메일, 사람 이름 등 개인·조직 식별 정보를 절대 적지 않는다. "실기기", "USB 연결 기기"처럼 일반화해서 쓴다. (이 규칙은 TODO.md만이 아니라 repo의 모든 파일에 적용 — CLAUDE.md 컨벤션 참고.)

## 사용 시점

- 작업 묶음 시작 전: 작업들을 펼쳐 적는다.
- 각 작업 완료 후: 체크하고 보고한다.
- 계획이 바뀌면: 목록을 수정한다.
